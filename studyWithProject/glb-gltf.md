# glb와 gltf의 차이

단일 파일(glb)이냐 아니냐(gltf)의 차이이다.

## 3개의 속성

- model.gltf (메타데이터 및 구조)
- model.bin (바이너리 데이터)
- texture.png (텍스처 이미지)

## 로드 방법

glb와 gltf는 거의 다르지 않다.

```
import { useGLTF } from "@react-three/drei";

function Model() {
  const { scene } = useGLTF("/path/to/model.gltf 혹은 glb");
  return <primitive object={scene} />;
}

export default Model;
```

## 깊게 알아보자

### GLTF (GL Transmission Format)

- 텍스트 기반: GLTF 파일은 JSON 형식으로 저장됩니다. 이는 사람이 읽기 쉽고 편집할 수 있습니다.
- 외부 리소스: 텍스처, 애니메이션, 셰이더 등 추가 리소스는 별도의 파일(예: .bin, 이미지 파일)로 저장됩니다.
- 파일 구조: GLTF 파일은 모델의 메타데이터와 구조 정보를 포함하고, 실제 데이터는 별도 파일로 저장됩니다. 예를 들어, .gltf 파일과 함께 .bin 파일 및 텍스처 이미지 파일들이 필요합니다.

### GLB (GLTF Binary)

- 이진 파일: 이진 형식으로 저장, 이는 사람이 읽기 어렵지만 컴퓨터가 더 효율적으로 읽음.
- 단일 파일: 모든 리소스를 하나의 파일에 포함합니다. 텍스처, 애니메이션, 셰이더 등을 포함한 모든 데이터를 단일 파일로 압축합니다.
- 파일 구조: GLTF 데이터를 포함한 바이너리 데이터가 결합된 하나의 파일로, 모델을 배포하고 사용하기에 더 편리

### GLTF와 GLB의 차이점 요약

| 차이점 | GLTF                      | GLB                                                       |
| ------ | ------------------------- | --------------------------------------------------------- |
| 포맷   | JSON 텍스트 형식          | 이진 형식                                                 |
| 구조   | 여러 파일로 나뉨          | 모든 데이터를 하나의 파일에 포함                          |
| 사용성 | 사람이 읽고 편집하기 쉬움 | 단일 파일로 관리가 편리하지만 사람이 읽고 편집하기 어려움 |
